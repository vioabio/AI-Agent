## 2026-07-01 | 基础通用代码配置

### 一、统一响应包装类

#### 1. `common/Result<T>` — 统一 API 响应体

- `code` / `message` / `data` / `timestamp` 四个字段
- 提供静态工厂方法：`success()`、`error()`
- 支持链式构造，兼容 `ResultCode` 枚举和自定义消息
- 使用 `@JsonInclude(NON_NULL)` 自动忽略空值

#### 2. `common/ResultCode` — 响应状态码枚举

| 分类 | 范围 | 示例 |
|------|------|------|
| HTTP 标准 | 200~503 | SUCCESS、BAD_REQUEST、UNAUTHORIZED、NOT_FOUND、INTERNAL_ERROR |
| 业务异常 | 1xxx | BUSINESS_ERROR(1000)、PARAM_ERROR(1001)、DATA_NOT_FOUND(1002) |
| 系统异常 | 2xxx | SYSTEM_ERROR(2000)、RPC_ERROR(2001)、DB_ERROR(2002) |

---

### 二、自定义异常

#### `exception/BusinessException` — 业务异常基类

- 继承 `RuntimeException`，携带 `code` 和 `message`
- 支持多种构造方式：自定义消息、`ResultCode`、带异常链
- 统一在 `GlobalExceptionHandler` 中捕获处理

---

### 三、全局异常处理器

#### `exception/GlobalExceptionHandler`

使用 `@RestControllerAdvice` 全局拦截异常，覆盖场景：

| 异常类型 | 处理方式 |
|----------|----------|
| `BusinessException` | 返回对应业务状态码和消息 |
| `MethodArgumentNotValidException` | 提取字段校验错误并拼接 |
| `BindException` | 提取表单绑定错误 |
| `ConstraintViolationException` | 提取约束校验错误 |
| `MissingServletRequestParameterException` | 提示缺少必要参数 |
| `MethodArgumentTypeMismatchException` | 提示参数类型错误 |
| `HttpMessageNotReadableException` | 提示请求体格式错误 |
| `HttpRequestMethodNotSupportedException` | 提示请求方法不支持 |
| `MaxUploadSizeExceededException` | 提示上传文件过大 |
| `NoHandlerFoundException` / `NoResourceFoundException` | 404 |
| `Exception` (兜底) | 500，日志记录完整堆栈 |

---

### 四、请求包装类

#### 1. `common/BasePageRequest` — 分页请求基类

- `pageNum`（默认1）、`pageSize`（默认10）、`sortField`、`sortOrder`
- 带 Knife4j `@Schema` 注解

#### 2. `common/PageResult<T>` — 分页响应包装

- `pageNum` / `pageSize` / `total` / `totalPages` / `records`
- 静态工厂 `of()` 自动计算总页数
- `empty()` 返回空分页结果

#### 3. `common/RequestWrapper` — 请求体缓存包装

- 继承 `HttpServletRequestWrapper`，缓存请求体字节
- 支持多次读取 `getInputStream()` / `getReader()`
- 提供 `getBody()` 直接获取请求体字符串
- 用于 Filter 中记录请求日志

#### 4. `common/RequestContext` — 请求上下文

- 基于 `ThreadLocal<Map<String, Object>>` 实现
- 存储 TraceId、UserId、Username、ClientIp、RequestUri
- 请求结束时必须调用 `clear()` 防止内存泄漏

---

### 五、全局跨域配置

#### `config/CorsConfig`

- 实现 `WebMvcConfigurer.addCorsMappings()`
- 注册 `CorsFilter` Bean 确保在 Spring Security Filter 之前处理
- `allowedOriginPatterns("*")` 允许所有来源（生产需改为具体域名）
- 允许所有常见 HTTP 方法和请求头，支持 Cookie 凭证
- 预检请求缓存 3600s

---

### 六、通用常量

#### `constant/Constants`

- 系统用户 ID、TraceId 请求头、日期格式常量等

## 2026-07-02 | Java 接入大模型的四种方式

### 依赖配置（`pom.xml`）

| 依赖 | 版本 | 用途 |
|------|------|------|
| `dashscope-sdk-java` | 2.19.1 | 阿里云灵积官方 SDK |
| `spring-ai-alibaba-starter` | 1.0.0-M6.1 | Spring AI 阿里云集成 |
| `langchain4j-community-dashscope` | 1.0.0-beta2 | LangChain4j 社区 DashScope 适配 |
| `hutool-all` | 5.8.37 | HTTP 工具类 |

---

### 一、SDK 接入（`demo/invoke/SdkAiInvoke.java`）

使用阿里云 DashScope 官方 SDK 直接调用：

- `Generation` + `GenerationParam` 构建请求
- 通过 `systemMsg` + `userMsg` 组装对话
- 模型：`qwen-plus`，输出格式：`MESSAGE`
- API Key 引用 `TestApiKey.API_KEY`

**运行方式**：单独运行 `SdkAiInvoke.main()`，不是在 Spring Boot 项目中运行。

---

### 二、HTTP 接入（`demo/invoke/HttpAiInvoke.java`）

通过 Hutool `HttpRequest` 直接请求阿里云 REST API：

- 端点：`https://dashscope.aliyuncs.com/api/v1/services/aigc/text-generation/generation`
- 手动构建 JSON 请求体（`model` + `input.messages` + `parameters`）
- `Authorization: Bearer <apiKey>` 认证

**运行方式**：单独运行 `HttpAiInvoke.main()`。

---

### 三、LangChain4j 接入（`demo/invoke/LangChainAiInvoke.java`）

使用 LangChain4j 社区版 DashScope 适配器：

- `QwenChatModel` 构建器配置 `apiKey` + `modelName("qwen-max")`
- 一行 `chat()` 即可调用，代码最简洁

**运行方式**：单独运行 `LangChainAiInvoke.main()`。

---

### 四、Spring AI 接入（`demo/invoke/SpringAiAiInvoke.java`）⚠ 未激活

通过 Spring AI Alibaba Starter 集成：

- 实现 `CommandLineRunner`，注入 `ChatModel dashscopeChatModel`
- `@Component` 注解**已被注释**，所以项目启动时**不会**自动执行
- 这是唯一一个设计为融入 Spring Boot 生命周期的调用方式

**运行方式**：取消 `@Component` 注释后，启动 Spring Boot 项目即自动执行。

---

### ⚠ 关键注意事项

1. **四种 demo 目前都是独立运行，未集成到 Spring Boot 应用中**。运行 `VioAiAgentApplication` 只会启动 Web 服务器，不会触发任何 AI 调用。要测试 AI 调用必须单独运行各自的 `main()` 方法。

2. **API Key 安全问题**：`TestApiKey.API_KEY` 硬编码在源码中，且已提交到 Git 仓库。建议迁移到 `application.yml` 配置文件并通过 `@Value` 注入，同时将 API Key 添加到 `.gitignore` 或使用环境变量。

3. **Spring AI Alibaba Starter 的 BOM 问题**：版本 `1.0.0-M6.1` 为里程碑版本，可能需要额外配置 Spring Milestone 仓库才能正常解析依赖。
