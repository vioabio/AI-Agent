## 一、什么是智能体？

**智能体（Agent）** 是一个能够感知环境、进行推理、制定计划、做出决策并自主采取行动以实现特定目标的 AI 系统。它以**大语言模型为核心**，集成**记忆、知识库和工具**等能力为一体，构造了完整的决策能力、执行能力和记忆能力。

与普通的 AI 大模型不同，智能体能够：

- **感知环境**：通过各种输入渠道获取信息（多模态），理解用户需求和环境状态
- **自主规划任务步骤**：将复杂任务分解为可执行的子任务，并设计执行顺序
- **主动调用工具完成任务**：根据需要选择并使用各种外部工具和 API，扩展自身能力边界
- **进行多步推理**：通过思维链（Chain of Thought）逐步分析问题并推导解决方案
- **持续学习和记忆过去的交互**：保持上下文连贯性，利用历史交互改进决策
- **根据环境反馈调整行为**：根据执行结果动态调整策略，实现闭环优化

### 智能体的分类

| 类型 | 特点 | 代表 |
|------|------|------|
| **反应式智能体** | 仅根据当前输入和固定规则做出反应，没有真正的规划能力 | 2023 年的 AI 聊天机器人 |
| **有限规划智能体** | 能进行简单的多步骤执行，但执行路径通常是预设的 | ChatGPT + Plugins |
| **自主规划智能体** | 根据任务目标自主分解任务、制定计划、选择工具并一步步执行 | Manus、AutoGPT、OpenManus |

## 二、智能体实现关键技术

### 2.1 CoT 思维链

**CoT（Chain of Thought）** 思维链是一种让 AI 像人类一样"思考"的技术，帮助 AI 在处理复杂问题时能够按步骤思考。

实现方式：
- 在 Prompt 中提供额外引导，如"让我们一步一步思考这个问题"
- 运用 few shot 技巧，给模型提供包含思维链的示例

**OpenManus 中的 CoT 系统提示词示例：**

```
You are an assistant focused on Chain of Thought reasoning. For each question, please follow these steps:

1. Break down the problem: Divide complex problems into smaller, more manageable parts
2. Think step by step: Think through each part in detail, showing your reasoning process
3. Synthesize conclusions: Integrate the thinking from each part into a complete solution
4. Provide an answer: Give a final concise answer

Your response should follow this format:
Thinking: [Detailed thought process, including problem decomposition, reasoning for each step, and analysis]
Answer: [Final answer based on the thought process, clear and concise]
```

### 2.2 Agent Loop 执行循环

**Agent Loop** 是智能体最核心的工作机制，指智能体在没有用户输入的情况下，自主重复执行推理和工具调用的过程。AI 回复后可能会继续自主执行后续动作，形成一个自主执行的循环，直到任务完成或超出预设的最大步骤数。

```java
public String execute() {
    List<String> results = new ArrayList<>();
    while (currentStep < MAX_STEPS && !isFinished) {
        currentStep++;
        String stepResult = executeStep();
        results.add("步骤 " + currentStep + ": " + stepResult);
    }
    if (currentStep >= MAX_STEPS) {
        results.add("达到最大步骤数: " + MAX_STEPS);
    }
    return String.join("\n", results);
}
```

### 2.3 ReAct 模式

**ReAct（Reasoning + Acting）** 是一种结合推理和行动的智能体架构，模仿人类解决问题时"思考 - 行动 - 观察"的循环：

- **推理（Reason）**：将原始问题拆分为多步骤任务，明确当前要执行的步骤
- **行动（Act）**：调用外部工具执行动作
- **观察（Observe）**：获取工具返回的结果，反馈给智能体进行下一步决策
- **循环迭代**：不断重复上述过程，直到任务完成或达到终止条件

```java
void executeReAct(String task) {
    String state = "开始";
    while (!state.equals("完成")) {
        String thought = "思考下一步行动";
        System.out.println("推理: " + thought);

        String action = "执行具体操作";
        System.out.println("行动: " + action);

        String observation = "观察执行结果";
        System.out.println("观察: " + observation);

        state = "完成";
    }
}
```

### 2.4 所需支持系统

1. **AI 大模型**：提供思考、推理和决策的核心能力
2. **记忆系统**：存储对话历史、中间结果和执行状态（Spring AI 的 `ChatMemory`）
3. **知识库**：通过 RAG 检索增强生成 + 向量数据库等技术，检索并利用专业知识
4. **工具调用**：扩展智能体能力边界的关键（包括 MCP）

**Compute Use**：一种特殊的工具调用，允许智能体直接与计算环境交互（执行代码、操作文件系统等），Claude 等平台已提供此类能力。

```
              └── Browser (浏览器操作 Agent)
```

### 4.2 核心实现

#### BaseAgent —— Agent Loop 实现

```python
class BaseAgent(BaseModel, ABC):
    async def run(self, request: Optional[str] = None) -> str:
        """执行代理的主循环"""
        if self.state != AgentState.IDLE:
            raise RuntimeError(f"Cannot run agent from state: {self.state}")

        if request:
            self.update_memory("user", request)

        results: List[str] = []
        async with self.state_context(AgentState.RUNNING):
            while (self.current_step < self.max_steps and
                  self.state != AgentState.FINISHED):
                self.current_step += 1
                step_result = await self.step()

                if self.is_stuck():
                    self.handle_stuck_state()

                results.append(f"Step {self.current_step}: {step_result}")

            if self.current_step >= self.max_steps:
                self.current_step = 0
                self.state = AgentState.IDLE
                results.append(f"Terminated: Reached max steps ({self.max_steps})")

        return "\n".join(results) if results else "No steps executed"

    @abstractmethod
    async def step(self) -> str:
        """执行单步操作，必须由子类实现"""
```

使用**模板方法设计模式**，父类定义执行流程，具体的执行方法（step）交给子类实现。

#### ReActAgent —— think + act

```python
class ReActAgent(BaseAgent, ABC):
    @abstractmethod
    async def think(self) -> bool:
        """处理当前状态并决定下一步行动"""

    @abstractmethod
    async def act(self) -> str:
        """执行决定的行动"""

    async def step(self) -> str:
        """执行单步：思考和行动"""
        should_act = await self.think()
        if not should_act:
            return "Thinking complete - no action needed"
        return await self.act()
```

#### ToolCallAgent —— 工具调用机制

```python
class ToolCallAgent(ReActAgent):
    """能够执行工具调用的代理类"""

    available_tools: ToolCollection = ToolCollection(
        CreateChatCompletion(), Terminate()
    )
    tool_choices: TOOL_CHOICE_TYPE = ToolChoice.AUTO
    special_tool_names: List[str] = Field(default_factory=lambda: [Terminate().name])

    async def think(self) -> bool:
        """处理当前状态并使用工具决定下一步行动"""
        if self.next_step_prompt:
            user_msg = Message.user_message(self.next_step_prompt)
            self.messages += [user_msg]

        response = await self.llm.ask_tool(
            messages=self.messages,
            system_msgs=([Message.system_message(self.system_prompt)]
                        if self.system_prompt else None),
            tools=self.available_tools.to_params(),
            tool_choice=self.tool_choices,
        )

        self.tool_calls = tool_calls = (
            response.tool_calls if response and response.tool_calls else []
        )
        # ... 处理响应和消息
        return bool(self.tool_calls or content)

    async def act(self) -> str:
        """执行工具调用并处理结果"""
        if not self.tool_calls:
            return self.messages[-1].content or "No content or commands to execute"

        results = []
        for command in self.tool_calls:
            result = await self.execute_tool(command)
            tool_msg = Message.tool_message(
                content=result,
                tool_call_id=command.id,
                name=command.function.name,
                base64_image=self._current_base64_image,
            )
            self.memory.add_message(tool_msg)
            results.append(result)

        return "\n\n".join(results)
```

### 4.3 关键实现细节

#### 工具系统设计

**BaseTool 抽象基类**：所有工具的统一接口

```python
class BaseTool(ABC, BaseModel):
    name: str
    description: str
    parameters: Optional[dict] = None

    async def __call__(self, **kwargs) -> Any:
        return await self.execute(**kwargs)

    @abstractmethod
    async def execute(self, **kwargs) -> Any:
        """执行工具的具体逻辑，由子类实现"""

    def to_param(self) -> Dict:
        """将工具转换为函数调用格式"""
        return {
            "type": "function",
            "function": {
                "name": self.name,
                "description": self.description,
                "parameters": self.parameters,
            },
        }
```

**Terminate 终止工具**：允许智能体自主决定何时结束任务

**ToolCollection 工具集合**：统一管理多个工具实例，实现可插拔性

```python
class ToolCollection:
    def __init__(self, *tools: BaseTool):
        self.tools = tools
        self.tool_map = {tool.name: tool for tool in tools}

    def to_params(self) -> List[Dict[str, Any]]:
        return [tool.to_param() for tool in self.tools]

    async def execute(self, *, name: str, tool_input: Dict[str, Any] = None) -> ToolResult:
        tool = self.tool_map.get(name)
        if not tool:
            return ToolFailure(error=f"Tool {name} is invalid")
        try:
            result = await tool(**tool_input)
            return result
        except ToolError as e:
            return ToolFailure(error=e.message)
```

#### MCP 协议支持

MCP 的本质就是工具调用。OpenManus 通过 `MCPClients` 类（继承自 `ToolCollection`）将 MCP 服务集成到现有工具系统中，动态创建 `MCPClientTool` 实例作为每个远程工具的代理。

#### Python 代码执行沙箱

使用独立进程隔离代码执行，实现超时机制防止无限循环，截获和处理所有异常，重定向标准输出。

#### 状态管理与上下文切换

使用 Python 的 `@asynccontextmanager` 实现安全的状态转换，即使在异常情况下也能正确恢复状态。

### 5.3 BaseAgent 实现

```java
@Data
@Slf4j
public abstract class BaseAgent {

    private String name;
    private String systemPrompt;
    private String nextStepPrompt;
    private AgentState state = AgentState.IDLE;
    private int maxSteps = 10;
    private int currentStep = 0;
    private ChatClient chatClient;
    private List<Message> messageList = new ArrayList<>();

    public String run(String userPrompt) {
        if (this.state != AgentState.IDLE) {
            throw new RuntimeException("Cannot run agent from state: " + this.state);
        }
        if (StringUtil.isBlank(userPrompt)) {
            throw new RuntimeException("Cannot run agent with empty user prompt");
        }

        state = AgentState.RUNNING;
        messageList.add(new UserMessage(userPrompt));

        List<String> results = new ArrayList<>();
        try {
            for (int i = 0; i < maxSteps && state != AgentState.FINISHED; i++) {
                int stepNumber = i + 1;
                currentStep = stepNumber;
                log.info("Executing step " + stepNumber + "/" + maxSteps);

                String stepResult = step();
                String result = "Step " + stepNumber + ": " + stepResult;
                results.add(result);
            }

            if (currentStep >= maxSteps) {
                state = AgentState.FINISHED;
                results.add("Terminated: Reached max steps (" + maxSteps + ")");
            }
            return String.join("\n", results);
        } catch (Exception e) {
            state = AgentState.ERROR;
            log.error("Error executing agent", e);
            return "执行错误" + e.getMessage();
        } finally {
            this.cleanup();
        }
    }

    public abstract String step();

    protected void cleanup() {
    }
}
```

关键设计点：
- `chatClient` 属性由调用方传入，不写死大模型
- `messageList` 属性维护消息上下文列表
- `state` 属性控制智能体执行流程

### 5.4 ReActAgent 实现

```java
@EqualsAndHashCode(callSuper = true)
@Data
public abstract class ReActAgent extends BaseAgent {

    public abstract boolean think();

    public abstract String act();

    @Override
    public String step() {
        try {
            boolean shouldAct = think();

            return act();

            e.printStackTrace();
```

### 5.5 ToolCallAgent 实现

三种实现方案：
1. 基于 Spring AI 工具调用能力，手动控制工具执行（推荐用于学习 ReAct 模式）
2. 基于 Spring AI 工具调用能力，简化调用流程（推荐快速实现）
3. 自主实现工具调用能力（不推荐，开发成本高）

**采用方案一，关键代码：**

```java
@EqualsAndHashCode(callSuper = true)
@Data
@Slf4j
public class ToolCallAgent extends ReActAgent {

    private final ToolCallback[] availableTools;
    private ChatResponse toolCallChatResponse;
    private final ToolCallingManager toolCallingManager;
    private final ChatOptions chatOptions;

    public ToolCallAgent(ToolCallback[] availableTools) {
        super();
        this.availableTools = availableTools;
        this.toolCallingManager = ToolCallingManager.builder().build();
        // 禁止 Spring AI 托管工具调用，自主实现
        this.chatOptions = DashScopeChatOptions.builder()
                .withProxyToolCalls(true)
                .build();
    }

    @Override
    public boolean think() {
        // 传入工具列表并调用大模型，得到需要调用的工具列表
        if (getNextStepPrompt() != null && !getNextStepPrompt().isEmpty()) {
            UserMessage userMessage = new UserMessage(getNextStepPrompt());
            getMessageList().add(userMessage);
        }
        List<Message> messageList = getMessageList();
        Prompt prompt = new Prompt(messageList, chatOptions);
        try {
            ChatResponse chatResponse = getChatClient().prompt(prompt)
                    .system(getSystemPrompt())
                    .tools(availableTools)
                    .call()
                    .chatResponse();

            this.toolCallChatResponse = chatResponse;
            AssistantMessage assistantMessage = chatResponse.getResult().getOutput();
            List<AssistantMessage.ToolCall> toolCallList = assistantMessage.getToolCalls();

            if (toolCallList.isEmpty()) {
                getMessageList().add(assistantMessage);
                return false;
            } else {
                return true;
            }
        } catch (Exception e) {
            log.error(getName() + "的思考过程遇到了问题: " + e.getMessage());
            return false;
        }
    }

    @Override
    public String act() {
        if (!toolCallChatResponse.hasToolCalls()) {
            return "没有工具调用";
        }

        Prompt prompt = new Prompt(getMessageList(), chatOptions);
        ToolExecutionResult toolExecutionResult =
                toolCallingManager.executeToolCalls(prompt, toolCallChatResponse);

        setMessageList(toolExecutionResult.conversationHistory());

        ToolResponseMessage toolResponseMessage =
                (ToolResponseMessage) CollUtil.getLast(toolExecutionResult.conversationHistory());

        // 检查是否调用了终止工具
        boolean terminateToolCalled = toolResponseMessage.getResponses().stream()
                .anyMatch(response -> "doTerminate".equals(response.name()));
        if (terminateToolCalled) {
            setState(AgentState.FINISHED);
        }
        // ... 返回结果
    }
}
```

## 六、扩展知识

### 6.1 智能体工作流

当面对复杂任务时，可以通过智能体工作流（Agent Workflow）编排多个专业智能体协同工作。

#### 五种工作模式

| 模式 | 说明 | 适用场景 |
|------|------|----------|
| **Prompt Chaining 提示链** | 将复杂任务拆解为有序子任务，逐步推进 | 内容生成（大纲 -> 内容 -> 润色） |
| **Routing 路由分流** | 根据输入类型将任务分发给最合适的智能体 | 客服系统、多模型系统 |
| **Parallelization 并行化** | 任务拆分为多个并行子任务，最后聚合结果 | 代码安全审查、长文档处理 |
| **Orchestrator-Workers 协调器-执行者** | 中央管理者动态拆解任务并分配给多个工人 | 复杂任务、不确定结构的任务 |
| **Evaluator-Optimizer 评估-优化** | 一个智能体生成，另一个评估反馈，循环迭代 | 机器翻译、质量打磨 |

#### 工作流编排框架

- **LangGraph**（LangChain 团队）：将工作流表示为状态转换图，支持动态分支、并行执行、思维链支持
- **Spring AI Alibaba Graph**：国内 Spring AI 生态的工作流编排框架，支持 ReAct Agent 模式和 Orchestrator-Workers 模式

**工作流编排示例（数据分析）：**

```java
public AIGraph<String, String> dataAnalysisWorkflow(
        PlanningAgent planningAgent,
        DataCollectionAgent dataCollectionAgent,
        AnalysisAgent analysisAgent,
        VisualizationAgent visualizationAgent,
        ReportingAgent reportingAgent) {

    return AIGraph.builder(String.class, String.class)
            .addNode("planning", input -> planningAgent.run(input),
                    Map.of("success", "data_collection"))
            .addNode("data_collection", input -> dataCollectionAgent.run(input),
                    Map.of("complete", "analysis", "incomplete", "planning"))
            .addNode("analysis", input -> analysisAgent.run(input),
                    Map.of("success", "visualization", "failure", "data_collection"))
            .addNode("visualization", input -> visualizationAgent.run(input),
                    Map.of("success", "reporting"))
            .addNode("reporting", input -> reportingAgent.run(input), Map.of())
            .withStartAt("planning")
            .build();
}
```

### 6.2 OWL 框架

OWL（Optimized Workforce Learning）是由 CAMEL-AI 团队开源的多智能体协作框架。主要特点：

- 多智能体协作：支持多个智能体之间的动态交互与协作
- 丰富的工具集成：浏览器自动化、代码执行、文档处理、音视频分析、搜索引擎等
- MCP 协议支持：与外部工具和数据源标准化对接
- 可定制与易用性：灵活配置和组合工具
- Web 可视化界面：基于 Gradio 的本地 Web UI

### 6.3 A2A 协议

**A2A（Agent to Agent）** 是智能体之间直接交流和协作的标准协议。

- **类比**：A2A 类似于智能体世界的 HTTP 协议
- **核心**：让每个智能体拥有自己的身份、能力描述和通信接口，包括身份认证、能力发现、任务委托、结果回传等机制
- **应用场景**：自动驾驶、制造车间、金融风控、智能客服等

**A2A 与 MCP 的区别：**

| 维度 | MCP | A2A |
|------|-----|-----|
| 作用对象 | 智能体与外部工具之间 | 智能体与智能体之间 |
| 类比 | "智能体-工具"的 HTTP 协议 | 智能体之间的通信协议 |
| 安全关注点 | 工具滥用和提示词注入攻击 | 身份认证、授权和信任链 |

对于一个成熟的智能体系统，MCP 负责智能体内部调用工具完成任务，A2A 负责智能体之间协同完成任务。

## 技术要点总结

| 要点 | 说明 |
|------|------|
| 智能体核心 | 大语言模型 + 记忆 + 知识库 + 工具 + 规划执行机制 |
| CoT 思维链 | 让 AI 按步骤思考，先思考后执行，展示推理过程 |
| Agent Loop | 智能体自主循环执行的核心机制，直到任务完成或达到最大步骤 |
| ReAct 模式 | 思考（Think）-> 行动（Act）-> 观察（Observe）的循环 |
| 模板方法模式 | 父类定义执行流程（BaseAgent），子类实现具体步骤（step/think/act） |
| 终止工具 | 让智能体自主决定何时结束任务，避免无限循环 |
| 工作流编排 | 多智能体协同：Prompt Chaining、Routing、Parallelization、Orchestrator-Workers、Evaluator-Optimizer |
| 开发建议 | 拥有自主规划能力的智能体虽然能完成复杂任务，但非常浪费 tokens，可能陷入无限循环，需按需使用 |
