## 一、第07章遗留 — 让 MCP 功能真正跑起来

当前 MCP 代码已全部完成，但需要以下手动操作才能验证功能：

| 序号 | 步骤 | 具体操作 | 优先级 |
|------|------|---------|--------|
| 1 | **注册 Pexels API Key** | 去 [pexels.com/api](https://www.pexels.com/api/) 免费注册，获取 Key | 🔴 高 |
| 2 | **编译 MCP Server** | `cd vio-image-search-mcp-server && mvnw clean package -DskipTests` | 🔴 高 |
| 3 | **设置 API Key** | 设置环境变量 `PEXELS_API_KEY=你的Key`，或修改 `mcp-servers.json` 中 `env` 字段 | 🔴 高 |
| 4 | **启动 MCP Server** | `java -jar vio-image-search-mcp-server/target/vio-image-search-mcp-server-0.0.1-SNAPSHOT.jar` | 🔴 高 |
| 5 | **重启主应用** | 重新启动 Spring Boot，MCP Client 自动连接 `localhost:8127` | 🔴 高 |
| 6 | **验证 MCP 图片搜索** | `curl "http://localhost:8123/api/ai/love_app/chat/mcp/sync?message=搜索可爱的猫咪图片&chatId=test123"` | 🔴 高 |
| 7 | **验证 MCP 流式调用** | `curl "http://localhost:8123/api/ai/love_app/chat/mcp/sse?message=搜索一张日落照片&chatId=test123"` | 🟡 中 |

### MCP 遗留问题

- **高德地图 MCP 依赖 Node.js**：`mcp-servers.json` 中 `amap-maps` 需要安装 Node.js 并注册高德 API Key。如不需要地图功能可暂时删除该配置项
- **Pexels API Key 不可硬编码**：当前通过 `System.getenv("PEXELS_API_KEY")` 获取，部署时需在环境中注入
- **MCP Server 启动顺序**：必须先启动 MCP Server，再启动主应用（SSE 模式）

## 二、第06章遗留 — API Key 和配置

| 序号 | 步骤 | 具体操作 | 优先级 |
|------|------|---------|--------|
| 8 | **注册 SearchAPI Key** | 去 [searchapi.io](https://www.searchapi.io) 注册免费账号，将获取的 Key 替换到 `application.yml` 中 `search-api.api-key` 的值 | 🔴 高 |

### 第06章遗留问题回顾

| 问题 | 说明 | 解决方案 |
|------|------|---------|
| PDF 中文字体依赖 | `STSongStd-Light` 字体在 Linux/Docker 下不存在 | 放入备用字体文件或安装 `fonts-chinese` |
| Terminal 工具 Windows 绑定 | 硬编码 `cmd.exe /c`，Linux/Mac 不可用 | 通过 `os.name` 判断系统自动切换 Shell |
| WebSearch 强依赖第三方 | SearchAPI 不可用时搜索功能完全失效 | 增加备用搜索 API |
| 工具安全风险 | Terminal 可执行任意命令，File 可读写任意路径 | 生产环境加命令白名单、限制操作目录 |
| 工具调用无重试机制 | 工具失败后 AI 可能陷入重试循环 | 后续通过 Agent ReAct 循环改善 |

### 第08章遗留问题回顾

| 问题 | 说明 | 改进方向 |
|------|------|---------|
| Token 消耗巨大 | Agent 每步调用一次 AI，多步任务成本是普通对话的数倍 | 简单对话用 LoveApp，复杂任务用 VioManus |
| 无长期记忆 | Agent 执行完毕后消息全部丢弃，无法连续对话 | 后续可集成 ChatMemory |
| 工具调用串行 | 多个无依赖的工具仍串行执行 | 后续可改造为并发执行 |
| 无 MCP 工具集成 | VioManus 只注入了本地 7 个工具，未集成 MCP 远程工具 | 在 VioManus 构造器中注入 `ToolCallbackProvider` |
| 无工作流编排 | 只有一个单体 Agent，无多智能体协作 | 后续引入 Spring AI Alibaba Graph |
| 无卡住/重复检测 | AI 可能陷入无效循环直到 maxSteps | 增加连续 N 步无进展检测 |

### 第09章遗留问题

| 问题 | 说明 | 解决方向 |
|------|------|---------|
| 前端未实际运行验证 | 代码已写完但未 `npm install` + `npm run dev` | 按步骤 16-19 操作 |
| Nginx 后端地址占位 | `nginx.conf` 中 `proxy_pass` 为 `http://your-backend-host:8123/api/` | 部署前替换为实际地址 |
| 无 Docker Compose | 前后端是独立容器，缺少统一编排（网络、依赖、环境变量） | 后续编写 `docker-compose.yml` |
| MCP Server 未纳入编排 | MCP Server 是独立 Java 项目，需单独部署或 STDIO 嵌入 | 部署时需额外处理 |
| 前端未挂载到 Git 父项目 | 前端是独立目录，有独立的 `.gitignore`，当前不在 Git 跟踪中 | 需手动 `git add vio-ai-agent-frontend/` |
