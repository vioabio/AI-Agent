# 06 - 工具调用

## 概述

本节以 Spring AI 框架为例，学习 AI 应用开发的核心特性 —— 工具调用，大幅增强 AI 的能力，并实战主流工具的开发，熟悉工具的原理和高级特性。

具体内容包括：

- 工具调用介绍
- Spring AI 工具开发
- 主流工具开发（文件操作、联网搜索、网页抓取、终端操作、资源下载、PDF 生成）
- 工具进阶知识（原理和高级特性）

---

## 核心知识点

- **工具调用（Tool Calling）**：让 AI 大模型借用外部工具来完成它自己做不到的事情
- **工具调用的实质**：AI 只负责决定何时需要用工具以及需要什么参数，真正执行工具的是应用程序
- **Spring AI 工具定义**：注解式（@Tool + @ToolParam）和编程式（MethodToolCallback）
- **ToolCallback 接口**：工具底层数据结构，包含 getToolDefinition()、getToolMetadata()、call() 方法
- **ToolCallingManager**：管理 AI 工具调用全过程的核心组件
- **立即返回（returnDirect）**：工具执行结果直接返回给用户，不再经过 AI 模型处理
- **ToolContext**：工具执行时传递额外上下文信息（如用户信息、会话 ID）
- **异常处理**：ToolExecutionExceptionProcessor 提供灵活的异常处理机制

## 一、需求分析

通过工具调用特性，可以实现以下需求：

1. **联网搜索**：智能推荐约会地点，示例："周末想带女朋友去上海约会，推荐几个适合情侣的小众打卡地？"
2. **网页抓取**：分析恋爱案例，示例："看看编程导航的其他情侣是怎么解决矛盾的？"
3. **资源下载**：下载恋爱相关的图片/音视频，示例："下载一张适合做手机壁纸的星空情侣图片"
4. **终端操作**：执行代码生成恋爱报告，示例："执行 Python 脚本来生成数据分析报告"
5. **文件操作**：保存用户恋爱档案，示例："帮我保存我的恋爱档案为文件"
6. **PDF 生成**：生成恋爱计划、情感分析报告 PDF，示例："生成一份《七夕约会计划》PDF"

这些需求还可以进行组合——用户先让 AI 联网搜索约会地点、再下载约会地点的图片、最后将获取到的内容组合生成 PDF 并保存到本地，一条龙服务。

## 二、工具调用介绍

### 什么是工具调用？

工具调用（Tool Calling）是让 AI 大模型借用外部工具来完成它自己做不到的事情。工具可以是网页搜索、对外部 API 的调用、访问外部数据、或执行特定的代码等。

### 工具调用的工作原理

AI 并不是自己调用工具，也不是把工具的代码发送给 AI 服务器让它执行。AI 只能提出要求，表示"我需要执行 XX 工具完成任务"。真正执行工具的是我们自己的应用程序，执行后再把结果告诉 AI，让它继续工作。

**完整流程：**

1. 用户提出问题
2. 程序将问题传递给大模型
3. 大模型分析问题，判断需要使用工具
4. 大模型输出工具名称和参数
5. 程序接收工具调用请求，执行工具操作
6. 工具执行并返回数据
7. 程序将结果传回给大模型
8. 大模型分析结果，生成最终回答
9. 程序将回答返回给用户

**为什么不让 AI 直接调用工具？** 关键原因是**安全性**——AI 模型永远无法直接接触你的 API 或系统资源，所有操作都必须通过程序来执行，这样可以完全控制 AI 能做什么、不能做什么。

### 工具调用和功能调用

Function Calling（功能调用）和 Tool Calling（工具调用）是同一概念，只是不同平台的叫法不同。

### 工具调用的技术选型

工具调用流程：

1. **工具定义**：程序告诉 AI"你可以使用这些工具"，描述每个工具的功能和参数
2. **工具选择**：AI 在对话中判断需要使用某个工具，准备相应参数
3. **返回意图**：AI 返回"我想用 XX 工具，参数是 XXX"
4. **工具执行**：程序接收请求，执行相应的工具操作
5. **结果返回**：程序将工具执行结果发回给 AI
6. **继续对话**：AI 根据工具返回的结果生成最终回答

建议使用 Spring AI、LangChain 等开发框架来简化开发。需要注意的是，不是所有大模型都支持工具调用。

## 三、Spring AI 工具开发

Spring AI 在实现工具调用时做的事情：

1. **工具定义与注册**：通过注解自动生成工具定义和 JSON Schema
2. **工具调用请求**：自动处理与 AI 模型的通信并解析工具调用请求，支持多工具链式调用
3. **工具执行**：提供统一的工具管理接口，自动解析参数进行调用
4. **处理工具结果**：内置结果转换和异常处理机制
5. **返回结果给模型**：封装响应结果并管理上下文
6. **生成最终响应**：自动整合工具调用结果到对话上下文

### 定义工具

#### 工具定义模式

Spring AI 提供两种模式：基于 Methods 方法（推荐）和基于 Functions 函数式编程。

| 特性 | Methods 方式 | Functions 方式 |
|------|------------|--------------|
| 定义方式 | 使用 @Tool 和 @ToolParam 注解 | 使用函数式接口通过 Spring Bean 定义 |
| 语法复杂度 | 简单，直观 | 较复杂 |
| 支持的参数类型 | 大多数 Java 类型 | 不支持基本类型、Optional、集合类型 |
| 支持的返回类型 | 几乎所有可序列化类型 | 不支持基本类型、Optional、集合类型 |

**Methods 模式示例：**

```java
class WeatherTools {
    @Tool(description = "Get current weather for a location")
    public String getWeather(@ToolParam(description = "The city name") String city) {
        return "Current weather in " + city + ": Sunny, 25°C";
    }
}

ChatClient.create(chatModel)
    .prompt("What's the weather in Beijing?")
    .tools(new WeatherTools())
    .call();
```

#### 注解式定义工具

```java
class WeatherTools {
    @Tool(description = "获取指定城市的当前天气情况")
    String getWeather(@ToolParam(description = "城市名称") String city) {
        return "北京今天晴朗，气温25°C";
    }
}
```

#### 编程式定义工具

```java
class WeatherTools {
    String getWeather(String city) {
        return "北京今天晴朗，气温25°C";
    }
}

Method method = ReflectionUtils.findMethod(WeatherTools.class, "getWeather", String.class);
ToolCallback toolCallback = MethodToolCallback.builder()
    .toolDefinition(ToolDefinition.builder(method)
            .description("获取指定城市的当前天气情况")
            .build())
    .toolMethod(method)
    .toolObject(new WeatherTools())
    .build();
```

**不支持作为工具方法参数或返回值的类型：** Optional、异步类型（CompletableFuture、Future）、响应式类型（Flow、Mono、Flux）、函数式类型（Function、Supplier、Consumer）

### 使用工具

#### 按需使用

```java
String response = ChatClient.create(chatModel)
    .prompt("北京今天天气怎么样？")
```

## 四、主流工具开发

开发注意事项：
- AI 自身能实现的功能没必要定义为工具（会增加额外交互）
- 工具返回值尽量使用 String 类型
- 格外注意工具描述的定义，影响 AI 决定是否使用工具

### 1. 文件操作

文件常量类：

```java
public interface FileConstant {
    String FILE_SAVE_DIR = System.getProperty("user.dir") + "/tmp";
}
```

文件操作工具类：

```java
public class FileOperationTool {

    private final String FILE_DIR = FileConstant.FILE_SAVE_DIR + "/file";

    @Tool(description = "Read content from a file")
    public String readFile(@ToolParam(description = "Name of the file to read") String fileName) {
        String filePath = FILE_DIR + "/" + fileName;
        try {
            return FileUtil.readUtf8String(filePath);
        } catch (Exception e) {
            return "Error reading file: " + e.getMessage();
        }
    }

    @Tool(description = "Write content to a file")
    public String writeFile(
        @ToolParam(description = "Name of the file to write") String fileName,
        @ToolParam(description = "Content to write to the file") String content) {
        String filePath = FILE_DIR + "/" + fileName;
        try {
            FileUtil.mkdir(FILE_DIR);
            FileUtil.writeUtf8String(content, filePath);
            return "File written successfully to: " + filePath;
        } catch (Exception e) {
            return "Error writing to file: " + e.getMessage();
        }
    }
}
```

### 2. 联网搜索

使用 Search API 实现从多个网站搜索内容。

```java
public class WebSearchTool {

    private static final String SEARCH_API_URL = "https://www.searchapi.io/api/v1/search";

    private final String apiKey;

    public WebSearchTool(String apiKey) {
        this.apiKey = apiKey;
    }

    @Tool(description = "Search for information from Baidu Search Engine")
    public String searchWeb(
            @ToolParam(description = "Search query keyword") String query) {
        Map<String, Object> paramMap = new HashMap<>();
        paramMap.put("q", query);
        paramMap.put("api_key", apiKey);
        paramMap.put("engine", "baidu");
        try {
            String response = HttpUtil.get(SEARCH_API_URL, paramMap);
            JSONObject jsonObject = JSONUtil.parseObj(response);
            JSONArray organicResults = jsonObject.getJSONArray("organic_results");
            List<Object> objects = organicResults.subList(0, 5);
            String result = objects.stream().map(obj -> {
                JSONObject tmpJSONObject = (JSONObject) obj;
                return tmpJSONObject.toString();
            }).collect(Collectors.joining(","));
            return result;
        } catch (Exception e) {
            return "Error searching Baidu: " + e.getMessage();
        }
    }
}
```

配置文件：

```yaml
search-api:
  api-key: 你的 API Key
```

### 4. 终端操作

通过 Java 的 Process API 实现终端命令执行。

**Windows 版本：**

```java
public class TerminalOperationTool {

    @Tool(description = "Execute a command in the terminal")
    public String executeTerminalCommand(@ToolParam(description = "Command to execute in the terminal") String command) {
        StringBuilder output = new StringBuilder();
        try {
            ProcessBuilder builder = new ProcessBuilder("cmd.exe", "/c", command);
            Process process = builder.start();
            try (BufferedReader reader = new BufferedReader(new InputStreamReader(process.getInputStream()))) {
                String line;
                while ((line = reader.readLine()) != null) {
                    output.append(line).append("\n");
                }
            }
            int exitCode = process.waitFor();
            if (exitCode != 0) {
                output.append("Command execution failed with exit code: ").append(exitCode);
            }
        } catch (IOException | InterruptedException e) {
            output.append("Error executing command: ").append(e.getMessage());
        }
        return output.toString();
    }
}
```

**依赖：**

```xml
    <artifactId>itext-core</artifactId>
```

**工具类：**

```java
public class PDFGenerationTool {

    @Tool(description = "Generate a PDF file with given content")
    public String generatePDF(
            @ToolParam(description = "Name of the file to save the generated PDF") String fileName,
            @ToolParam(description = "Content to be included in the PDF") String content) {
        String fileDir = FileConstant.FILE_SAVE_DIR + "/pdf";
        String filePath = fileDir + "/" + fileName;
        try {
            FileUtil.mkdir(fileDir);
            try (PdfWriter writer = new PdfWriter(filePath);
                 PdfDocument pdf = new PdfDocument(writer);
                 Document document = new Document(pdf)) {
                PdfFont font = PdfFontFactory.createFont("STSongStd-Light", "UniGB-UCS2-H");
                document.setFont(font);
                Paragraph paragraph = new Paragraph(content);
                document.add(paragraph);
            }
            return "PDF generated successfully to: " + filePath;
        } catch (IOException e) {
            return "Error generating PDF: " + e.getMessage();
        }
    }
}
```

```java
public class ToolRegistration {
```

## 五、工具进阶知识

### 工具底层数据结构

ToolCallback 接口是所有工具实现的基础：

```java
public interface ToolCallback {

    ToolDefinition getToolDefinition();

    ToolMetadata getToolMetadata();

    String call(String toolInput);

    String call(String toolInput, ToolContext tooContext);
}
```

- `getToolDefinition()`：提供工具的基本定义（名称、描述和调用参数），传递给 AI 模型
- `getToolMetadata()`：提供处理工具的附加信息
- `call()`：工具的执行入口

当使用注解定义工具时，Spring AI 的幕后工作：
- JsonSchemaGenerator 解析方法签名和注解，自动生成 JSON Schema 参数定义
- ToolCallResultConverter 将各种类型的方法返回值统一转换为字符串
- MethodToolCallback 封装注解方法，使其符合 ToolCallback 接口规范

### 工具上下文（ToolContext）

工具执行时传递额外上下文信息（如登录用户信息、会话 ID），这些信息不会传递给 AI 模型，只在应用程序内部使用。

**传递上下文参数：**

```java
String response = chatClient
        .prompt("帮我查询用户信息")
        .tools(new CustomerTools())
        .toolContext(Map.of("userName", "yupi"))
        .call()
        .content();
```

**在工具中使用上下文：**

```java
class CustomerTools {

    @Tool(description = "Retrieve customer information")
    Customer getCustomerInfo(Long id, ToolContext toolContext) {
        return customerRepository.findById(id, toolContext.get("userName"));
    }
}
```

**适用场景：**
- 用户认证信息：在上下文中传递用户 token，不暴露给模型
- 请求追踪：在上下文中添加请求 ID，便于日志追踪和调试
- 自定义配置：根据不同场景传递特定配置参数

### 立即返回（returnDirect）

工具执行结果直接返回给用户，不再经过 AI 模型处理。适合返回二进制数据（图片/文件）、大量数据不需要 AI 解释的工具、明确结果的操作（如数据库操作）。

**注解方式：**

```java
class CustomerTools {
    @Tool(description = "Retrieve customer information", returnDirect = true)
    Customer getCustomerInfo(Long id) {
        return customerRepository.findById(id);
    }
}
```

**编程方式：**

```java
ToolMetadata toolMetadata = ToolMetadata.builder()
    .returnDirect(true)
    .build();

Method method = ReflectionUtils.findMethod(CustomerTools.class, "getCustomerInfo", Long.class);
ToolCallback toolCallback = MethodToolCallback.builder()
    .toolDefinition(ToolDefinition.builder(method)
            .description("Retrieve customer information")
            .build())
    .toolMethod(method)
    .toolObject(new CustomerTools())
    .toolMetadata(toolMetadata)
    .build();
```

### 工具底层执行原理

#### ToolCallingManager

ToolCallingManager 是管理 AI 工具调用全过程的核心组件，负责根据 AI 模型的响应执行对应的工具并返回执行结果给大模型。

核心方法：
- `resolveToolDefinitions`：从模型的工具调用选项中解析工具定义
- `executeToolCalls`：执行模型请求对应的工具调用

使用 Spring AI Starter 时会默认初始化 DefaultToolCallingManager。

#### 框架控制的工具执行（默认模式）

- 框架自动检测模型是否请求调用工具
- 自动执行工具调用并获取结果
- 自动将结果发送回模型
- 管理整个对话流程直到得到最终答案

#### 用户控制的工具执行

通过设置 `internalToolExecutionEnabled` 为 false 来禁用内部工具执行：

```java
ChatOptions chatOptions = ToolCallingChatOptions.builder()
    .toolCallbacks(ToolCallbacks.from(new WeatherTools()))
    .internalToolExecutionEnabled(false)
    .build();

ToolCallingManager toolCallingManager = DefaultToolCallingManager.builder().build();

Prompt prompt = new Prompt("获取编程导航的热门项目教程", chatOptions);
ChatResponse chatResponse = chatModel.call(prompt);

while (chatResponse.hasToolCalls()) {
    ToolExecutionResult toolExecutionResult = toolCallingManager.executeToolCalls(prompt, chatResponse);
    prompt = new Prompt(toolExecutionResult.conversationHistory(), chatOptions);
    chatResponse = chatModel.call(prompt);
}

System.out.println(chatResponse.getResult().getOutput().getText());
```

### 异常处理

ToolExecutionExceptionProcessor 接口：

```java
@FunctionalInterface
public interface ToolExecutionExceptionProcessor {
    String process(ToolExecutionException exception);
}
```

默认实现类 DefaultToolExecutionExceptionProcessor 提供两种策略：
- `alwaysThrow = false`：将异常信息作为错误消息返回给 AI 模型
- `alwaysThrow = true`：直接抛出异常，中断当前对话流程

**自定义异常处理器：**

```java
@Bean
ToolExecutionExceptionProcessor customExceptionProcessor() {
    return exception -> {
        if (exception.getCause() instanceof IOException) {
            return "Unable to access external resource. Please try a different approach.";
        } else if (exception.getCause() instanceof SecurityException) {
            throw exception;
        }
        return "Error executing tool: " + exception.getMessage();
    };
}
```

### 工具解析（ToolCallbackResolver）

```java
public interface ToolCallbackResolver {
    @Nullable
    ToolCallback resolve(String toolName);
}
```

Spring AI 默认使用 DelegatingToolCallbackResolver，将解析任务委托给：
- SpringBeanToolCallbackResolver：从 Spring 容器中查找工具
- StaticToolCallbackResolver：从预先注册的 ToolCallback 列表中查找

**自定义解析器：**

```java
@Bean
ToolCallbackResolver toolCallbackResolver(List<ToolCallback> toolCallbacks) {
    StaticToolCallbackResolver staticResolver = new StaticToolCallbackResolver(toolCallbacks);

    ToolCallbackResolver customResolver = toolName -> {
        if (toolName.startsWith("dynamic-")) {
            return createDynamicTool(toolName.substring(8));
        }
        return null;
    };

    return new DelegatingToolCallbackResolver(List.of(customResolver, staticResolver));
}
```

## 技术要点总结

1. **工具调用让 AI 从"大脑"变成"有手有脚"的智能体**，能完成 AI 自身无法直接完成的任务
2. **安全性设计**：AI 只负责判断和决策，程序负责执行，所有操作可控
3. **Spring AI 通过注解（@Tool + @ToolParam）简化工具定义**，自动处理 JSON Schema 生成和结果转换
4. **六大主流工具**：文件操作、联网搜索、网页抓取、终端操作、资源下载、PDF 生成
5. **工具注册模式**集中管理工具，包含工厂模式、依赖注入模式、适配器模式
6. **ToolCallingManager 是工具调用的核心**：框架控制模式（默认）和用户控制模式（更灵活）
7. **returnDirect 立即返回**适合二进制数据和不需要 AI 解释的场景
8. **ToolContext 传递安全上下文**，不暴露给 AI 模型，增强了灵活性和安全性
9. **异常处理**可通过 ToolExecutionExceptionProcessor 灵活定制
10. **ToolCallbackResolver** 支持运行时动态解析工具，适合从数据库搜索工具的复杂场景
