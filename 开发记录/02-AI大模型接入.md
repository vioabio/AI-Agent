# 02 - AI 大模型接入

## 一、概述

本节从 AI 大模型上手，学会如何使用 AI 大模型以及通过编程调用 AI 大模型。具体内容包括：

- AI 大模型概念
- 接入 AI 大模型（3 种方式）
- 后端项目初始化
- 程序调用 AI 大模型（4 种方式）
- 本地部署 AI 大模型
- Spring AI 调用本地大模型

## 二、核心知识点

### 2.1 什么是 AI 大模型？

AI 大模型是指具有超大规模参数（通常为数十亿到数万亿）的深度学习模型，通过对大规模数据的训练，能够理解、生成人类语言，处理图像、音频等多种模态数据，并展示出强大的推理和创作能力。

**涌现能力**：随着模型参数量和训练数据量的增加，模型会展现出训练过程中未明确赋予的新能力，比如逻辑推理、代码编写、多步骤问题解决等。

**主流大模型举例**：

| 厂商 | 模型 | 特点 |
|------|------|------|
| OpenAI | GPT-4o、GPT-4、GPT-3.5 Turbo | 多模态支持 |
| Anthropic | Claude 3 系列（Opus/Sonnet/Haiku） | 由强到弱 |
| Google | Gemini Ultra/Pro/Nano | 多模态能力 |
| Meta | Llama 3（70B/8B）、Llama 2 | 开源 |
| 百度 | 文心一言 | 国产 |
| 阿里 | 通义千问 | 国产 |
| 字节跳动 | 豆包 | 国产 |
| 科大讯飞 | 星火 | 国产 |

### 2.2 AI 大模型的分类

**1、按模态分类**

- **单模态模型**：仅处理单一类型数据（如早期的 GPT-3）
- **多模态模型**：能处理多种类型信息
  - 文本 + 图像：GPT-4V、Gemini、Claude 3
  - 文本 + 音频 + 视频：GPT-4o

**2、按开源性分类**

- **闭源模型**（GPT-4、Claude、Gemini）：通过 API 访问，付费使用
- **开源模型**（Llama 系列、Mistral、Falcon）：可本地部署，自由调整，但性能通常略逊

**3、按规模分类**

- **超大规模**（数千亿到数万亿参数）：如 GPT-4 (1.76T)，需要大量计算资源
- **中小规模**（几十亿到几百亿参数）：如 Llama 3 (70B)、Mistral 7B，可在普通硬件上运行

**4、按用途分类**

- **通用模型**：GPT-4、Claude 3、Gemini
- **特定领域模型**：医疗（Med-PaLM 2）、代码（CodeLlama、StarCoder）、科学（Galactica）

### 2.3 开发者学习重点

作为开发者，不需要深入了解大模型的实现原理（如 Transformer 架构、注意力机制），而应专注于：

1. **如何选择适合的 AI 大模型**：考虑应用场景、成本与性能平衡、数据隐私要求
2. **如何有效利用 AI**：接入方式（API/SDK/框架）、Prompt Engineering、大模型开发工作流
3. **了解模型能力边界**：知道模型能做什么和不能做什么、幻觉问题及处理方法

### 2.4 大模型选择对比维度

| 维度类别 | 具体评估点 |
|----------|-----------|
| 功能支持 | 多模态能力、工具使用能力、上下文窗口大小、指令遵循能力 |
| 性能指标 | 准确性、响应质量、知识时效性 |
| 部署与集成 | 部署方式、API 接口、并发处理能力 |
| 商业与合规 | 成本效益、数据安全与隐私、法律合规性 |
| 生态与支持 | 社区支持、文档完善度、技术支持 |

**对大多数开发者来说，更关注的是：准确度 + 功能支持 + 性能 + 成本。**

## 三、开发步骤

### 步骤 1：接入 AI 大模型（2 种途径）

#### 途径一：云服务

直接使用云服务商在云端已部署好的大模型服务，无需考虑基础设施：

- 提供纯净的大模型能力和构建应用（智能体）的工具
- 按需付费，无需前期大量基础设施投入
- 随时可用，维护成本低
- 自动更新到最新版本
- 更完善的安全措施和合规保障

#### 途径二：自部署

开发者自行在本地或私有云环境部署开源大模型：

- 完全控制数据流，更高的数据隐私保障
- 可根据特定需求微调和定制模型
- 无网络延迟，适合对响应速度有严格要求的场景
- 一次性成本较高，需要专业的技术团队维护

> **建议**：个人开发者选用云服务即可；对数据安全有严格要求的企业适合自部署。

### 步骤 2：选择接入方式（3 种方式）

#### 方式一：AI 应用平台接入

以 **阿里云百炼** 为例，这是一站式的大模型开发及应用构建平台，支持在 5 分钟内开发出一款大模型应用。

支持创建三种应用类型：
- **智能体应用**：对话式 AI 助手
- **工作流应用**：编排多个工作节点完成复杂任务，适合高确定性的业务流程
- **智能体编排应用**：通过画布自定义智能体执行逻辑，实现复杂多智能体协同

> 注意区分百炼（可视化平台，更上层）和灵积 DashScope（模型 API 接口，更底层）。编程调用 AI 大模型时更多与灵积打交道。

#### 方式二：AI 软件客户端接入

- **Cherry Studio**：集多模型对话、知识库管理、AI 绘画、翻译等功能于一体的全能 AI 助手平台
- **Cursor**：以 AI 为核心的编程开发工具，可快速生成项目代码、理解整个代码库并提供智能建议

#### 方式三：程序接入

分为两种方式：
1. **直接调用 AI 大模型**（更原生）：使用 SDK/API 或 AI 开发框架（Spring AI、LangChain4j）
2. **调用 AI 大模型平台创建的应用**（更方便）：使用特定平台 SDK/API

> 个人小项目推荐方式 2；企业级项目推荐方式 1，直接用 Spring AI 等开发框架。

### 步骤 3：后端项目初始化

### 步骤 4：程序调用 AI 大模型（4 种方式）

#### 方式一：SDK 接入

**引入依赖**（pom.xml）：

```xml
<dependency>
    <groupId>com.alibaba</groupId>
    <artifactId>dashscope-sdk-java</artifactId>
    <version>2.19.1</version>
</dependency>
```

**示例代码**：

```java
import com.alibaba.dashscope.aigc.generation.Generation;
import com.alibaba.dashscope.aigc.generation.GenerationParam;
import com.alibaba.dashscope.aigc.generation.GenerationResult;
import com.alibaba.dashscope.common.Message;
import com.alibaba.dashscope.common.Role;
import com.alibaba.dashscope.utils.JsonUtils;

public class SdkAiInvoke {

    public static GenerationResult callWithMessage() throws Exception {
        Generation gen = new Generation();
        Message systemMsg = Message.builder()
                .role(Role.SYSTEM.getValue())
                .content("You are a helpful assistant.")
                .build();
        Message userMsg = Message.builder()
                .role(Role.USER.getValue())
                .content("你是谁？")
                .build();
        GenerationParam param = GenerationParam.builder()
                .apiKey(TestApiKey.API_KEY)
                .model("qwen-plus")
                .messages(Arrays.asList(systemMsg, userMsg))
                .resultFormat(GenerationParam.ResultFormat.MESSAGE)
                .build();
        return gen.call(param);
    }

    public static void main(String[] args) {
        try {
            GenerationResult result = callWithMessage();
            System.out.println(JsonUtils.toJson(result));
        } catch (Exception e) {
            System.err.println("An error occurred: " + e.getMessage());
        }
    }
}
```

#### 方式二：HTTP 接入

适合 SDK 不支持的编程语言或需要更灵活控制的场景。

```java
public class HttpAiInvoke {
    public static void main(String[] args) {
        String url = "https://dashscope.aliyuncs.com/api/v1/services/aigc/text-generation/generation";

        Map<String, String> headers = new HashMap<>();
        headers.put("Authorization", "Bearer " + TestApiKey.API_KEY);
        headers.put("Content-Type", "application/json");

        JSONObject requestBody = new JSONObject();
        requestBody.put("model", "qwen-plus");

        JSONObject input = new JSONObject();
        JSONObject[] messages = new JSONObject[2];

        JSONObject systemMessage = new JSONObject();
        systemMessage.put("role", "system");
        systemMessage.put("content", "You are a helpful assistant.");
        messages[0] = systemMessage;

        JSONObject userMessage = new JSONObject();
        userMessage.put("role", "user");
        userMessage.put("content", "你是谁？");
        messages[1] = userMessage;

        input.put("messages", messages);
        requestBody.put("input", input);

        JSONObject parameters = new JSONObject();
        parameters.put("result_format", "message");
        requestBody.put("parameters", parameters);

        HttpResponse response = HttpRequest.post(url)
                .addHeaders(headers)
                .body(requestBody.toString())
                .execute();

        if (response.isOk()) {
            System.out.println("请求成功，响应内容：" + response.body());
        } else {
            System.out.println("请求失败，状态码：" + response.getStatus());
        }
    }
}
```

#### 方式三：Spring AI（Spring AI Alibaba）

**引入依赖**（pom.xml）：

```xml
<dependency>
    <groupId>com.alibaba.cloud.ai</groupId>
    <artifactId>spring-ai-alibaba-starter</artifactId>
    <version>1.0.0-M6.1</version>
</dependency>
```

**添加仓库配置**（pom.xml）：

```xml
<repositories>
  <repository>
    <id>spring-milestones</id>
    <name>Spring Milestones</name>
    <url>https://repo.spring.io/milestone</url>
    <snapshots>
      <enabled>false</enabled>
    </snapshots>
  </repository>
</repositories>
```

**application.yml 配置**：

```yaml
spring:
  application:
    name: spring-ai-alibaba-qwq-chat-client-example
  ai:
    dashscope:
      api-key: ${AI_DASHSCOPE_API_KEY}
      chat:
        options:
          model: qwen-plus
```

**示例代码**：

```java
@Component
public class SpringAiAiInvoke implements CommandLineRunner {

    @Resource
    private ChatModel dashscopeChatModel;

    @Override
    public void run(String... args) throws Exception {
        AssistantMessage output = dashscopeChatModel.call(new Prompt("你好，我是鱼皮"))
                .getResult()
                .getOutput();
        System.out.println(output.getText());
    }
}
```

> ChatModel 适合简单对话场景；ChatClient 提供更多高级功能（如会话记忆），适合复杂场景。

### 步骤 5：接入方式对比

| 接入方式 | 优点 | 缺点 | 适用场景 |
|----------|------|------|----------|
| SDK 接入 | 类型安全、编译时检查、完善的错误处理 | 依赖特定版本、可能增加项目体积 | 需要深度集成、单一模型提供商 |
| HTTP 接入 | 无语言限制、不增加额外依赖、灵活性高 | 手动处理错误、代码冗长 | SDK 不支持的语言、简单原型验证 |
| Spring AI | 统一抽象接口、易于切换模型、与 Spring 生态融合 | 增加额外抽象层、版本快速迭代 | Spring 应用、需要支持多种模型 |
| LangChain4j | 完整的 AI 工具链、支持复杂工作流 | 学习曲线较陡、文档相对较少 | 构建复杂 AI 应用、RAG 应用开发 |

> **推荐选择 Spring AI**：属于 Spring 生态更主流，简单易用、资源更多，满足绝大多数 AI 项目开发需求。

### 步骤 6：本地部署和接入 AI 大模型

#### 6.1 使用 Ollama 安装本地大模型

1. 下载安装 Ollama 及其命令行工具
2. 在 Ollama 模型广场挑选模型（建议先选择小模型）
3. 执行命令安装并运行：

```bash
ollama run gemma3:1b
```

4. 安装成功后可在终端直接对话，访问 `http://localhost:11434` 确认运行状态。

#### 6.2 Spring AI 调用 Ollama 大模型

**引入依赖**（pom.xml）：

```xml
<dependency>
    <groupId>org.springframework.ai</groupId>
    <artifactId>spring-ai-ollama-spring-boot-starter</artifactId>
    <version>1.0.0-M6</version>
</dependency>
```

**application.yml 配置**：

```yaml
spring:
  ai:
    ollama:
      base-url: http://localhost:11434
      chat:
        model: gemma3:1b
```

**示例代码**：

```java
public class OllamaAiInvoke implements CommandLineRunner {

    @Resource
    private ChatModel ollamaChatModel;

    @Override
    public void run(String... args) throws Exception {
        AssistantMessage output = ollamaChatModel.call(new Prompt("你好，我是鱼皮"))
                .getResult()
                .getOutput();
        System.out.println(output.getText());
    }
}
```

## 四、技术要点总结

1. **大模型核心是涌现能力**：模型规模越大，能力越强，同时成本也越高。
2. **云服务 vs 自部署**：个人开发者选云服务，企业敏感数据选自部署。
3. **Spring AI 是最佳实践**：统一抽象接口、易于切换模型、与 Spring 生态完美融合。
4. **ChatModel 适合简单场景，ChatClient 适合复杂场景**：ChatClient 支持链式调用、会话记忆、Advisor 等高级功能。
5. **Ollama 是本地部署首选**：简化了大模型安装和环境配置，支持命令行和 API 调用。
6. **API Key 安全管理**：生产环境中应使用配置文件或环境变量，不要硬编码在代码中。
7. **先测试后开发**：无论选择哪种接入方式，都建议先用简单测试案例验证接入是否成功。
